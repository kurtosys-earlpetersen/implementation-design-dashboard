Overview

Figma Styles to InDesign — Figma plugin + InDesign script. Extracts paragraph, character, color, and effect styles from Figma and imports into InDesign. Supports merge when updating existing docs so styles refresh without duplication.

Import/Export vs Create-and-Edit

Export styles from Figma → Import in InDesign. One flow; styles created in bulk. Without it: manually creating each style in InDesign and matching Figma by hand. Recommendation: Use import/export.

Time Savings & Speed

~5–8 hours per design system (vs manual creation of 20–50+ styles). ~10–50× speed increase.

1. Overview

Dynamic Type Scale Accessibility Tool for checking font sizing, unit conversions, and WCAG 2.1 contrast and readability across Web, Print, and Mobile. Supports font selection, type scale generation, DPI settings, and export (XML, JSON, CSV, JSX, IDJS).

3. Type Scale

Seven-level scale (XS–3XL). XS ~8.2px FAIL, SM ~10.2px FAIL, BASE ~12.8px FAIL, LG 16px PASS, XL 20px PASS, 2XL 25px PASS, 3XL ~31.3px PASS.

4. Accessibility

Contrast: AA 4.5:1, AAA 7:1. Font size: Minimum 16px for body text. Exports support design and dev workflows including InDesign XML import.

Deep Dive — K2-9226 Delivery Flow

Tonderai Matanga
Current State Problems
Linear, handover-heavy process
Design introduced too late in cycle
Multiple approval loops create bottlenecks
Fragmented ownership between teams
Future State — 3-Phase Model
Phase 1
Discovery & Feasibility
Map requirements, assess feasibility, align stakeholders
Phase 2
Design & Alignment
Build-ready design inputs, clear ownership, structured collaboration
Phase 3
Build & Validation
Configuration, QA, UAT with defined gates (DoR / DoD)
