Overview

Figma Styles to InDesign — Figma plugin + InDesign script. Main files: FigmaStylesToInDesign.jsx, FigmaPlugin/code.js, ui.html. Extracts paragraph, character, color, and effect styles from Figma (JSON or .jsx) and imports into InDesign. Supports merge when updating existing docs so styles refresh without duplication. Keeps design system in Figma and InDesign in sync.

Import/Export vs Create-and-Edit

Export styles from Figma (JSON or .jsx) → Import in InDesign. One flow; styles created in bulk from design source. Without it: manually creating each paragraph style, character style, swatch, and object style in InDesign and matching Figma by hand. Recommendation: Use import/export. Creating and editing dozens of styles by hand is slow and error-prone.

Time Savings & Speed

~5–8 hours per design system (vs manual creation and sync of 20–50+ styles). ~10–50× speed increase (bulk creation in seconds/minutes vs hours).

Effective Usage

Use import/export for any design-system or template work. Use “Merge” when updating an existing doc so styles refresh without duplication. Keeps brand consistency and avoids re-creating styles by hand.

Summary

Import/export (Figma → InDesign) is the most impactful for design-system and template work. Using the script as intended maximizes time savings and keeps workflows predictable.

3. Type Scale Specimens

Seven-level type scale (XS–3XL) with specimen text, size (px), weight, and Accessibility Check (PASS/FAIL). Scale levels: XS ~8.2px FAIL, SM ~10.2px FAIL, BASE ~12.8px FAIL (3.2px below 16px body min), LG 16.0px PASS, XL 20.0px PASS, 2XL 25.0px PASS, 3XL ~31.3px PASS. Base unit conversions: px, pt, em/rem, vw/vh/% (context-dependent).

4. Accessibility Checks

Contrast: AA 4.5:1 (normal text), AAA 7:1 (normal text). Large text: AA 3:1, AAA 4.5:1. Large text = 18pt (24px) or 14pt (~18.66px) bold and larger. Font size: Minimum 16px for body text. Font weight: 400 appropriate. Display options: “Show Accessibility Checks” and “Use WCAG Standards”.

5. Standards Referenced

Web: Desktop 1920×1080 (Full HD); DPI 96; base font 16px (1rem); 1vw = 1% width. Print: US Letter 816×1056px @ 96dpi; A4 793×1122px; 1pt = 1/72 inch; body 10–12pt. Mobile: iPhone/Android viewports; base font 16px minimum; touch targets 44×44px minimum.

6. Font & Scale Controls

Font source: e.g. Inter; upload custom (.ttf, .otf, .woff, .woff2). Type scale: Major Third (1.25); base size configurable; Regenerate. Colors: foreground and background for contrast analysis.

7. DPI Behavior

Context DPI for Web/Desktop, Print, Mobile (96, 72, 144, 192). Formula: pixels = (points ÷ 72) × DPI. 96 DPI: 1pt = 1.333px; 72 DPI: 1pt = 1px; 144+ DPI: retina/high-DPI.

8. Export

Formats: XML, JSON, CSV, JSX, IDJS. Context: Web, Print, or Mobile. Optional: color swatches, character/paragraph styles. InDesign: File → Import XML.

9. Summary for Report

The tool provides type scale specimens, unit conversions (px, pt, rem, em, vw, vh, %), and WCAG 2.1–oriented checks across Web, Print, and Mobile. Export supports design and dev workflows including InDesign XML import.

1. Overview

Kleen is an InDesign document cleanup script that removes clutter and inconsistencies in a single run. It targets empty frames, unused styles and swatches, hidden objects, local overrides, and duplicate swatches. The script offers a checklist so you choose which actions to run, plus Unused and Duplicates views to review items before applying. Optional save/load presets support repeated cleanup workflows. One undo reverts the entire run.

2. What Kleen Cleans

TargetDescription
Empty framesFrames with no content (graphic or text)
Unused stylesParagraph, character, object, table, and cell styles not applied in the document
Unused swatchesColor swatches not used by any object, style, or table
Hidden objectsObjects on the pasteboard or set to hidden
OverridesLocal formatting overrides (e.g. on paragraph or character style)
Duplicate swatchesSwatches that match existing colors; can merge into one

3. How It Works

Checklist: Enable or disable each cleanup type. Unused view: Lists unused styles/swatches before deletion. Duplicates view: Lists duplicate swatches and target for merge. Run: Executes only checked actions. Undo: One InDesign Undo reverts the full Kleen run. Presets: Save checklist state and reload (e.g. “Pre-delivery”, “Template cleanup”).

4. When to Use

Before final delivery or archive; after major edits; template hygiene. With Find Unused Styles: Kleen can remove unused styles; Find Unused Styles is style-focused and explicit. Use both as needed.

5. Time Savings & Speed

Estimated time savings: ~20–60 min per document. Approx. speed increase: ~5–15× (one cleanup run vs many manual steps).

6. Summary for Report

Kleen automates InDesign document cleanup. A checklist and Unused/Duplicates views give control over what is removed or merged; one undo reverts all. Use for pre-delivery, post-edit, and template hygiene. Complements Find Unused Styles for style-only cleanup.
